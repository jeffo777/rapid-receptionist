# SEO & GEO Reference — Rapid Receptionist

## Overview

Rapid Receptionist maintains both traditional SEO infrastructure (sitemap, robots.txt, JSON-LD schema) and a Generative Engine Optimisation (GEO) layer for AI search engines (ChatGPT, Gemini, Claude, Perplexity). This document maps every file, explains data flow, and provides validation steps.

## JSON-LD Knowledge Graph

All JSON-LD entities use `@id` anchors and cross-reference each other:

```
Organization (@id: /#organization)
  ├── WebSite (@id: /#website) → publisher: /#organization
  ├── Service (TradeHub) (@id: /plumbing-contractors/ai-receptionist/#service) → provider: /#organization
  ├── Service (ChannelTrade) (@id: /plumbing-contractors/ai-phone-answering/#service) → provider: /#organization
  └── Service (ChannelService) (@id: /construction-trades/ai-phone-answering/#service) → provider: /#organization
```

Every Service schema links back to the Organization via `"provider": {"@id": "https://rapidreceptionist.co.uk/#organization"}`. This creates a connected knowledge graph rather than isolated data blobs.

## Data Flow

### Pricing Data Flow

```
pricing.json (source of truth)
  ├── Page .astro files → import pricing → inject into FAQs, CTAs, subheads
  ├── Layout files → auto-generate pricing notes, CTA defaults, Service schema offers
  │
  │   ⚠️ MANUAL SYNC REQUIRED for these files:
  ├── public/rapid-facts.json → pricing section
  ├── public/.well-known/rapid-manifest.json → products array
  ├── public/llms.txt → pricing references
  └── public/llms-full.txt → pricing section
```

### Trade Data Flow

```
trades.json (source of truth)
  ├── Page .astro files → trade-specific content (manual, unique per page)
  ├── Layout files → auto-generate trade grids, breadcrumbs
  ├── Homepage index.astro → auto-generate trade directory
  │
  │   ⚠️ MANUAL SYNC REQUIRED for these files:
  ├── public/rapid-facts.json → tradesServed.count
  ├── public/.well-known/rapid-manifest.json → tradesServed section
  ├── public/llms.txt → trades list
  └── public/llms-full.txt → trades list with vocabulary
```

### Channel Data Flow

```
channels.json (source of truth)
  ├── Layout files → auto-generate channel feature grids, pricing notes
  ├── Homepage index.astro → auto-generate channel cards
  │
  │   ⚠️ MANUAL SYNC REQUIRED for these files:
  ├── public/rapid-facts.json → channels section
  ├── public/.well-known/rapid-manifest.json → products array
  ├── public/llms.txt → channel list
  └── public/llms-full.txt → channel capability framework
```

## Validation Steps

Run these checks after any changes to SEO/GEO files:

### 1. Build Check
```bash
npx astro build
```
Must complete with 0 errors.

### 2. JSON Validation
Validate all JSON files are syntactically correct:
```powershell
Get-Content public/rapid-facts.json | ConvertFrom-Json | Out-Null
Get-Content public/.well-known/rapid-manifest.json | ConvertFrom-Json | Out-Null
```
No errors = valid JSON.

### 3. JSON-LD Spot Check
After build, open a built HTML file and search for `application/ld+json` to verify:
- Organization has `@id: /#organization`
- WebSite has `@id: /#website` and `publisher` reference
- Service schemas have `@id` and `provider` reference to `/#organization`

### 4. Cross-File Pricing Check
Verify pricing consistency across all files:
- `pricing.json` → £49 core, £23 add-on, £295 setup, 120 mins, £0.20 overage
- `rapid-facts.json` → same numbers
- `rapid-manifest.json` → same numbers
- `llms.txt` → same numbers
- `llms-full.txt` → same numbers

### 5. Trade Count Check
Verify the trade count is consistent:
- `trades.json` → count the array length
- `rapid-facts.json` → `tradesServed.count`
- `rapid-manifest.json` → `tradesServed.count`
- `llms.txt` → "46" in intro and section headers
- `llms-full.txt` → "46" in intro
- BaseLayout schema → "46" in Organization description
